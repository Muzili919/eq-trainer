# EQ Trainer 产品定义

## 一句话

**用 AI 实时生成无限场景，训练徐志胜/黄渤式幽默高情商聊天能力的 Web 应用。**

## 目标用户

- **首要用户**：项目作者（自己天天用，先把自己练成 case 0）
- **延伸用户**：想提升聊天技能但读书学不会、苦于"实战机会少"的人
- **不是给谁**：想学销售话术的、想学辩论的、想 PUA 的

## 核心问题

1. 看书看不会高情商聊天，因为没场景练
2. 真实场景**偶发**，等不来——必须靠"模拟无限训练"补上
3. 高情商**没有标准答案**，靠死记被动反应，离应用最远
4. 市面没人教"幽默高情商"，因为难——这正好是产品差异化

## 核心循环（每日训练）

```
打开应用
   ↓
[1] 今日推送（AI 实时生成 3-5 道场景题）
    依据：薄弱技能（SRS）+ 历史风格 + 难度梯度
    例："你刚升职，前同事说『终于轮到你了，我们这种没背景的就这命』"
   ↓
[2] 用户回应
    打字（MVP）/ 语音（后置）
   ↓
[3] AI 灵活判断（4 维度 + 描述性反馈）
    得体度 / 化解力 / 幽默指数 / 风格匹配度
    不打"对错"，给"为什么 + 可以怎么改"
   ↓
[4] 苏格拉底教学（如果不到 70 分）
    AI 反问 1-3 轮 → 用户重写
    3 轮还不行 → 给标准示范 + 引用风格基准（"黄渤在 X 场景这样说过：...")
   ↓
[5] 入熟练度系统
    本次涉及的技能进度更新，影响明日推送
```

## 入口结构

| 入口 | 类型 | 频率 | 说明 |
|---|---|---|---|
| **每日训练** | 主 | 必经 | 打开就有题，AI 推 3-5 道，10 分钟搞定 |
| **真实场景日记** | 辅 | 偶发 | 遇到真实困境时主动来记，AI 帮诊断+改写 |
| **自由对练** | 辅 | 想玩就玩 | 选场景类型/对手类型，无限对话不限轮 |
| **技能树** | 辅 | 可视化 | 看进度、看薄弱点，可主动挑技能专项练 |

**关键决策**：真实场景**不当主入口**，因为素材偶发；当辅助入口，有素材就来一笔。

## 关键差异化（护城河）

1. **幽默作为核心维度**：评分独立打"幽默指数"，市面没人这么做
2. **真人风格基准**：徐志胜/黄渤/李雪琴/何炅作为 few-shot 锚点，AI 评价/示范都向他们靠
3. **苏格拉底教学法**：不直接给答案，反问引导——这才能内化成判断力
4. **AI 实时生成场景 + 实时判断**：无限新鲜，无标准答案束缚
5. **吉祥物"星海" + 视觉差异化**：手感纸质 + 暖紫 + 微插画，反"AI 冷感"，做成有人情味的学习陪伴 App

## 输入方式与商业化定位

| 阶段 | 输入 | 定位 |
|---|---|---|
| **MVP（V2，自用）** | 文字 + **按住录音 STT**（浏览器 Web Speech API） | 自用免费 |
| **开源免费版** | 文字（保留语音输入） | GitHub 开源，谁部署谁用 |
| **付费版（远期）** | 完整**语音对讲**（端到端实时语音轮回 + TTS 风格化回应） | 这是付费墙，难度和成本对得起付费 |

**护城河逻辑**：免费版能跑通核心训练循环（文字+STT 已经够"即兴"），付费版价值点是"像跟真人说话练"——这个体验差异足以让真心想练的人愿意付费。

**为什么 MVP 不做付费**：先把产品做透，自己天天用，用 1-3 个月跑通核心循环再考虑付费墙。

## 吉祥物：星海（导师 IP）

紫色小海马造型 + 蝴蝶翅膀 + 抱着发光卷轴，源自"星光影海兽"图集左上角第一张（抱书海马）。

**角色定位**：
- 苏格拉底导师 — 它问问题，不直接给答案
- 全产品贯穿 — 反问气泡、空状态、打卡庆祝都有它
- 后期开源宣传的核心视觉资产

**官方名待定**：工作代号"星海"。

## MVP 范围（2-3 周一版）

✅ 必做：
- 登录/注册（本地账号，OAuth 后置）
- 技能树：12 个核心技能（详见 `SKILLS.md`）
- 每日训练（AI 生成场景 + AI 判断 + 苏格拉底引导 + 风格匹配）
- 输入：文字 + 按住录音 STT（V2）
- 真实场景日记（语音/文字输入 + AI 苏格拉底诊断 + 链接技能）
- SRS 简化版（5 级熟练度，间隔重复）
- 每日打卡 + 连续天数
- 吉祥物"星海"贯穿（欢迎语、苏格拉底反问、空状态、打卡庆祝）

❌ 后置：
- 周复盘视图
- 雷达图/数据分析（首页迷你版有，专门页 V2 后置）
- 完整语音对话（V3）
- 商业化（API key 计费、订阅）
- 多端同步
- 朋友对战 / 排行榜
- 自由对练（MVP 阶段先简化为只能 8 轮内）

## UI 流程（页面级）

**设计原则：移动端优先**。桌面端只是大屏适配（max-width 居中布局），核心交互按手机设计。原因：高情商训练是碎片时间用的——通勤、午休、排队，都是手机场景。

```
[未登录]
  └─ Login（登录/注册二合一）

[已登录]
  ├─ Home（每日训练入口 + 推荐技能 + 紧凑打卡）  ← 主页
  ├─ Practice（练习场景 - 单题对话）
  │    └─ Result（练习结果 - 评分+反馈+苏格拉底）
  ├─ Diary（真实场景日记列表 + 新建）
  │    └─ DiaryDetail（单篇 + AI 诊断 + 微练习入口）
  ├─ Skills（技能树视图，可点击进入专项）
  │    └─ SkillDetail（单个技能 - 定义/示例/进度/题）
  └─ Profile（个人 - 连续天数/总练习数/技能统计）
```

**关于打卡日历**：弱化，不占主区。Home 页放小卡片即可（最近 7 天 + 一行 streak 数字），完整月历搬到 Profile 页或干脆不做。

## 成功指标（DAU 之前先看这个）

**给自己用的阶段（前 3 个月）**：
- 我自己每天用的次数（目标：日均 1 次以上 = 形成习惯）
- 连续打卡天数（目标：第一个月跑通 21 天）
- 主观感受：跟人聊天时是否能更快想到合适的回应（每周自评 1-10）
- 真实场景日记数（目标：每周 ≥ 2 篇，证明生活中有在"用上"）

**对外开源后看**：
- GitHub Star
- 自部署用户数
- 打赏 / 订阅转化（B1 商业化阶段）

## 不做什么（反向定义）

- ❌ 不做"心理咨询"——情商训练 ≠ 心理治疗
- ❌ 不做"恋爱话术"——目标不是 PUA
- ❌ 不做"销售话术"——目标不是说服别人买单
- ❌ 不做"辩论训练"——目标不是辩赢
- ✅ 做的是"让对话舒服、让关系顺、让自己幽默不油腻"
