## EQ Trainer Bug 修复

修复了两个线上问题：

### 问题1：AI 情绪标签显示英文 "expectant"
- **根因**：P1 场景生成 prompt 中 `ai_emotion` 有 `expectant`（期待）等英文枚举值，前端直接原文展示
- **修复**：PracticePage.tsx 添加 `EMOTION_LABELS` 映射表，将 `expectant` → "期待"等
- **文件**：`frontend/src/pages/PracticePage.tsx`

### 问题2：日记页面 Internal Server Error / Bad Gateway
- **真正的根因**：`NOT NULL constraint failed: diary.their_words`
  - `initiate`（我开口）模式下前端传 `their_words: ""`
  - 路由代码 `body.their_words or None` 把空字符串转成了 `None`
  - 数据库表 `diary.their_words` 是 `VARCHAR NOT NULL`，插入 None 直接炸
  - SQLModel 的 `create_all` 不会 ALTER 已有表的列约束
- **次要因素**：
  1. 后端无全局异常处理，DB 错误直接导致裸 500
  2. `diary.py` 的 `create_diary` 无 try/except
  3. LLM 客户端无超时设置
- **修复**：
  1. **`db.py`**：添加 `_migrate_nullable_columns()` 迁移函数，自动 ALTER TABLE 让 `their_words` 允许 NULL（含 SQLite 重建表方案）
  2. **`diary.py`**：`their_words` 空字符串不再转 None；`diagnose_diary()` 加 try/except 兜底
  3. **`main.py`**：添加全局异常处理器，返回友好错误消息
  4. **`llm.py`**：给 OpenAI 客户端添加 `timeout=60s` + `httpx.AsyncClient(timeout=60s)`
  5. **`DiaryCreate`** 所有字段加 `default=""`
  6. **`api.ts`** 添加 502/504 超时友好提示
  7. **`diary.py`** `list_diaries` 端点 `d.context` 加 None 保护
- **文件**：`backend/app/core/db.py`, `backend/app/main.py`, `backend/app/api/v1/diary.py`, `backend/app/services/llm.py`, `frontend/src/lib/api.ts`

### 踩坑
- `httpx.AsyncHTTPClient` 不存在，正确的是 `httpx.AsyncClient`
- SQLModel `create_all()` 不会修改已有表的列约束，需要手动 ALTER TABLE
- SQLite 不支持 `ALTER COLUMN`，需要用重建表方式（RENAME → CREATE → INSERT → DROP）
- 路由中 `or None` 对空字符串的危险性：空字符串 `""` 在 Python 中是 falsy，`"" or None` 变成 `None`
